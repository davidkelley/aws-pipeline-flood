    public handler(event: Object, context: Object, callback: Function) source 

    import {handler} from 'pipeline-flood/functions/steps/create/create.js'

    This handler will parse an event object, download files specified and then upload them (POST) to flood.io.

    Flood.io specific options will also be parsed from the event and passed along. Required parameters are:

    • tool, string, e.g. 'jmeter'
    • grids, object
      • region, string, e.g. 'eu-west-1
      • instance_type, string, e.g. 'm4.xlarge'
      • stop_after, string, e.g. '60'
      • infrastructure, string, e.g. 'demand
    • files, array

    See: https://github.com/flood-io/api-docs/blob/master/endpoints/floods/POST-floods.md

    Callback function will be called with an object describing the flood session on success

    {
 flood: {
   uuid, string
 },
 grid: {
   uuid, string
 },
 status, string, e.g. 'queued'/'finished'
}

    NameTypeAttributeDescription
    event Object

    object containing list of test files to be executed and flood io options

    callback function that will receive either (err) or (null, response)

    public transformObjectProps(obj: Object, parent: Object, pre: String): * source 

    import {transformObjectProps} from 'pipeline-flood/functions/steps/helper.js'

    Attempts to remap an object structure to a form compatible object. Takes an object like

    {
 flood: {
   tool: 'jmeter'
   grids: [
     {
       region: 'eu-west-1'
     }
   ]
 }
}

    and transforms it to

    {
 'flood[tool]': 'jmeter',
 'flood[grids][][region]': 'eu-west-1'
}

    Params:

    NameTypeAttributeDescription
    obj Object

    object containing the flood io preferences
    parent Object

    parent object to insert into (used for recursion)
    pre String

    prefix, will be generated from keys if not provided
